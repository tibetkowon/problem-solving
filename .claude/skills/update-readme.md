# README 자동 업데이트 스킬

## Trigger

**모든 코딩 작업이 끝날 때마다** 사용자 요청 없이 반드시 실행한다:
- 기능/API/컴포넌트 구현 완료 직후
- 버그 수정 완료 직후
- 의존성·환경변수·설정 변경 직후
- "완료", "커밋할게", "다음 기능으로" 신호 직전

README 업데이트 없이 작업을 완료 선언하지 않는다.

## 언어 규칙

**README는 반드시 한국어(한글)로 작성한다.** 기술 용어(Spring Boot, JWT, API 등)는 영어 그대로 사용하되, 설명 문장과 섹션 제목은 모두 한글로 작성한다.

---

## 실행 절차

### 1단계: 현재 상태 파악

- `README.md` 존재 여부 확인
- 방금 완료한 작업의 성격 파악
- 빌드 도구·기술 스택 확인 (pom.xml, build.gradle, package.json 등)

### 2단계: README 없는 경우 → 신규 생성

프로젝트에 맞게 아래 구조로 **한국어**로 작성한다:

```markdown
# [프로젝트명]
> 한 줄 설명

## 기술 스택

## 시작하기

### 사전 요구사항

### 설치 및 실행
```bash
# 실행 명령어
```

## 주요 기능

## API (해당하는 경우)
| 메서드 | 경로 | 설명 | 응답 코드 |
|--------|------|------|-----------|

## 환경변수 (해당하는 경우)
| 변수명 | 설명 | 예시 |
|--------|------|------|

## 프로젝트 구조
```

### 3단계: README 있는 경우 → 변경 사항만 반영

기존 구조와 스타일을 유지하면서 변경된 내용과 관련된 섹션만 업데이트한다.

| 작업 종류 | 업데이트 섹션 |
|-----------|--------------|
| 새 API 엔드포인트 | API 섹션에 행 추가 |
| 새 기능 | 기능 목록에 항목 추가 |
| 기능 수정/제거 | 관련 섹션 수정 |
| 새 의존성 | 기술 스택, 사전 요구사항 업데이트 |
| 환경변수 추가 | 환경변수 섹션 업데이트 |
| 실행 방법 변경 | 실행 섹션 업데이트 |

**지켜야 할 원칙:**
- 기존 내용을 불필요하게 삭제하지 않는다
- 코드에서 확인된 사실만 기록한다 (추측 금지)
- README 전체를 재작성하지 않는다 — 변경된 부분만 수정

### 4단계: 완료 보고

```
README.md 업데이트 완료:
- [변경 내용 요약]
```

## Spring Boot 프로젝트 API 문서 형식

```markdown
## API

### [도메인명]
| 메서드 | 경로 | 설명 | 요청 바디 | 응답 |
|--------|------|------|-----------|------|
| POST | /api/users | 사용자 생성 | `{"name":"...","email":"..."}` | 201 |
| GET | /api/users/{id} | 사용자 조회 | - | 200 |
```

## Node.js 프로젝트 환경변수 형식

```markdown
## 환경변수

`.env` 파일을 프로젝트 루트에 생성:

| 변수명 | 설명 | 예시 |
|--------|------|------|
| `PORT` | 서버 포트 | `3000` |
| `MONGODB_URI` | MongoDB URI | `mongodb://...` |
| `JWT_SECRET` | JWT 시크릿 | 임의의 긴 문자열 |
```

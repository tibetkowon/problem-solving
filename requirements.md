## 기능 요구사항
1. 단원 조회
	1. 단원 조회
		- 단원 목록을 조회한다.
2. 단원별 문제 풀이
	1. 랜덤 문제조회
		- 사용자가 단원을 선택하면 해당 단원의 문제를 **하나씩** 제공한다.
		- 현재 객관식 문제 선택지는 총 5개 주어진다. (현재는 5지선다)
		- 문제는 복수 정답/단일 정답 문제가 따로 존재한다.
		- 사용자가 풀지 않은 문제에서 문제를 1개 랜덤 조회한다.
		- '문제 넘기기' 버튼을 누르면 현재 문제를 풀지 않고 같은 챕터 내의 새로운 문제를 랜덤으로 제공받는다. 단, 직전에 건너뛴 문제는 다음 문제 랜덤 추출 대상에서 제외된다.
		    - 특정 사용자의 특정 chapter에 대해 바로 직전에(1회 전) 건너뛴 문제 제외
		- 정답률 계산
		    - 동일 문제를 30명 이상 사용자가 해당 문제를 푼 경우에만 해당 문제에 대한 평균 정답률 데이터를 제공한다.
		    - 부분 정답의 경우 정답률 계산에서는 틀린 것으로 간주한다.
		    - 30명 이상 푼 문제: 소수점 첫째 자리에서 반올림 (예: 66.7% → 67%)
		    - 30명 미만: `null` 반환
		    **정답률 계산 로직의 효율성 중요!**
		- 해당 단원의 모든 문제를 다 풀었거나 더 이상 제공할 문제가 없는 경우의 예외 처리(Exception Handling) 또는 적절한 응답 코드를 고민하여 구현해 주세요
		- request 예시
			```json
			{ 
				"chapterId": 1,
				"userId": 1,
			}
			```
		- response 예시
		```json
		{ 
			"problemId": 1, 
			"content": "문제 설명", 
			"choices": ["지문1", "지문2", ...], 
			"answerCorrectRate": 23.
		}
		```

	1. 문제 제출
		- 객관식: 선택한 선택지 번호들 제출 (복수 정답 가능)
		- 주관식: 텍스트로 답안 제출
		- 사용자가 제출한 답에 따라 정답/오답/복수 정답이 가능하다.
		    - 부분 정답이 가능하다
		        - 오답 포함 여부와 상관없이 정답을 1개라도 포함했다면 부분 정답입니다
		            - \[1,2] 가 정답일 때 \[1,3]을 제출해도 부분 정답 처리 된다.
		- 제출 즉시 정답 여부와 해설이 반환 된다.
		- request 예시
			```json
			{ 
				"problemId": 1, 
				"userId": 1,
				"answerType": "주관식", 
				"userAnswer": ... <- 타입 기호에 맞게 설정
			}
			```
		- response 예시
			```json
			{ 
				"problemId": 1, 
				"answerStatus": "correct", // 정답, 부분 정답, 오답 
				"explaination": "문제 해설...", 
				"problemAnswers": ... <- 타입 기호에 맞게 설정
			}
			```
3. 풀었던 문제
	1. 풀었던 문제 조회
		1. 풀었던 문제 목록
			- 사용자가 풀었던 문제의 목록을 조회할 수 있다.
		2. 풀었던 문제 상세
			- 사용자는 자신이 풀었던 단원별 문제 풀이를 상세 조회할 수 있다.
			- request 예시
		```json
		{ 
			"userId": 1, 
			"problemId": 3
		}
		```
		- response 예시
		```json
		{ 
			"problemId": 1, 
			"problemStatus": ... <- 문제 타입,
			"answerStatus": "correct", // 정답, 부분 정답, 오답 
			"explaination": "문제 해설...", 
			"problemAnswers": ... <- 타입 기호에 맞게 설정, 
			"userAnswers": ... <- 타입 기호에 맞게 설정, 
			"answerCorrectRate" : 23
		}
		```

## 객체
- 단원
	ID - PK
	단원명
- 문제
	ID - PK
	단원ID
	문제내용
	문제타입
	문제해설
	문제정답(주관식용)
	*index: 단원ID*
- 문제 선택지 (객관식용)
	ID - PK
	문제ID
	선택지내용
	정답여부
	*index: 문제 ID*
- 회원-문제 (문제제출)
	ID - PK
	회원ID
	문제ID
	회원 제출 주관식답변
	*index: 회원ID_문제ID*
- 회원-문제-선택지 (객관식 답변)
	ID - PK
	회원-문제ID
	회원 제출 객관식 선택지
	*index: 회원-문제ID*
- 문제 정답률(백업용)
	문제ID - PK
	제출수
	정답수
## 상수
- 문제 타입
	- 객관식(단일 정답)
	- 객관식(복수 정답)
	- 주관식
- 문제 응답 타입
	- 정답
	- 부분정답
	- 오답

#### 정답율 계산 플로우
- 정답 제출 -> 정답여부 확인 -> redis 저장(문제:응답수, 문제:정답수)
- 문제 조회(상세/풀이) ->  redis 조회 -> 정답율 계산 -> 응답
- redis 데이터 백업 -> 5분 주기로 redis 의 데이터들(문제, 응답수, 정답수) DB 백업
